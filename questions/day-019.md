# SQL Scenario-Based Interview Question: Identify High-Churn-Risk SaaS Subscribers

## Business Scenario
You are a Data Analyst at 'ProTrack Analytics', a SaaS company offering project management and analytics tools. The Customer Success team is concerned about subscriber churn and wants to proactively engage with users who might be "ghost subscribers" – those who are paying for an active subscription but are not actively using the product.

Your task is to identify all currently active subscribers whose last recorded product activity (e.g., login, creating a project, viewing a report) was more than 30 days ago. This list will be used for a targeted re-engagement campaign.

Assume:
```sql
CURRENT_DATE = '2023-10-26'
```

# Database Setup

## Create Users Table
```sql
CREATE TABLE Users (
    user_id VARCHAR(50) PRIMARY KEY,
    username VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    created_at DATE NOT NULL
);
```

## Create Subscriptions Table
```sql
CREATE TABLE Subscriptions (
    subscription_id VARCHAR(50) PRIMARY KEY,
    user_id VARCHAR(50) NOT NULL,
    plan_name VARCHAR(100) NOT NULL,
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    status VARCHAR(50) NOT NULL, -- e.g., 'active', 'cancelled', 'trial_expired'
    FOREIGN KEY (user_id) REFERENCES Users(user_id)
);
```

# Task
Write a SQL query that returns the `user_id`, `username`, `email`, `plan_name`, `subscription_end_date`, `last_activity_date`, and the `days_inactive` for all users who meet the following criteria:
1.  They have an 'active' subscription as of `CURRENT_DATE`. An active subscription means `status = 'active'` AND `end_date >= CURRENT_DATE`.
2.  Their last recorded product activity was more than 30 days prior to `CURRENT_DATE`. Users with no activity logs should also be included and considered inactive.

Order the results by `days_inactive` in descending order.

---

# Expected Output Format
| user_id | username   | email                   | plan_name  | subscription_end_date | last_activity_date | days_inactive |
| :------ | :--------- | :---------------------- | :--------- | :-------------------- | :----------------- | :------------ |
| U005    | EveDavis   | eve.davis@example.com   | Pro Plan   | 2024-04-19            | NULL               | 9999          |
| U002    | BobJohnson | bob.johnson@example.com | Basic Plan | 2023-11-09            | 2023-08-16         | 71            |

*Note: The `days_inactive` for `U005` can be represented as a very large number, or calculated based on their `created_at` or `subscription_start_date` if `last_activity_date` is `NULL` and a specific business rule exists. For simplicity, `NULL` or a placeholder like `9999` is acceptable for calculation purposes, or a simple `NULL` is fine if the column can be NULLable for `days_inactive`.*
*The specific calculation for `days_inactive` when `last_activity_date` is `NULL` should be discussed during the interview to clarify assumptions. Here, we assume it counts as max inactivity.*

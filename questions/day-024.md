# SQL Scenario-Based Interview Question: Calculating Monthly SaaS Revenue with Plan Changes

## Business Scenario
A SaaS company offers various subscription plans to its customers. Customers have the flexibility to upgrade or downgrade their plans at any point during their monthly billing cycle. When a plan change occurs mid-month, the customer is billed pro-rata for the respective days spent on each plan. The finance department needs an accurate calculation of the total billed amount for each customer for a specific calendar month, which is crucial for revenue recognition and accurate invoicing. As a Data Analyst, you are tasked with generating this report.

Assume:
```sql
CURRENT_DATE = '2023-03-15'
```

---

# Database Setup

## Create Customers Table
```sql
CREATE TABLE Customers (
    customer_id INT PRIMARY KEY,
    customer_name VARCHAR(255)
);
```

## Insert Customers Data
```sql
INSERT INTO Customers (customer_id, customer_name) VALUES
(1, 'Alice'),
(2, 'Bob'),
(3, 'Charlie');
```

## Create SubscriptionPlans Table
```sql
CREATE TABLE SubscriptionPlans (
    plan_id INT PRIMARY KEY,
    plan_name VARCHAR(255),
    monthly_price DECIMAL(10, 2)
);
```

## Insert SubscriptionPlans Data
```sql
INSERT INTO SubscriptionPlans (plan_id, plan_name, monthly_price) VALUES
(101, 'Basic', 50.00),
(102, 'Pro', 100.00),
(103, 'Premium', 200.00);
```

## Create CustomerSubscriptionHistory Table
```sql
CREATE TABLE CustomerSubscriptionHistory (
    subscription_history_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT,
    plan_id INT,
    effective_start_date DATE,
    effective_end_date DATE, -- NULL if the plan is currently active
    FOREIGN KEY (customer_id) REFERENCES Customers(customer_id),
    FOREIGN KEY (plan_id) REFERENCES SubscriptionPlans(plan_id)
);
```

## Insert CustomerSubscriptionHistory Data
```sql
INSERT INTO CustomerSubscriptionHistory (subscription_history_id, customer_id, plan_id, effective_start_date, effective_end_date) VALUES
(1, 1, 101, '2023-01-01', '2023-03-10'), -- Alice: Basic Plan until March 10 (inclusive)
(2, 1, 102, '2023-03-11', NULL),          -- Alice: Pro Plan from March 11 onwards

(3, 2, 102, '2023-02-15', NULL),          -- Bob: Pro Plan from Feb 15 onwards

(4, 3, 101, '2023-02-01', '2023-03-05'), -- Charlie: Basic Plan until March 5 (inclusive)
(5, 3, 102, '2023-03-06', '2023-03-20'), -- Charlie: Pro Plan from March 6 to March 20 (inclusive)
(6, 3, 103, '2023-03-21', NULL);         -- Charlie: Premium Plan from March 21 onwards
```

# Task
Calculate the total billed amount for each customer for the calendar month of **March 2023**.
The calculation should account for pro-rata billing based on the number of days a customer was on each plan within that specific month.

Assumptions for calculation:
- The target month is March 2023, which has 31 days.
- A plan's `effective_start_date` and `effective_end_date` are both inclusive.
- If `effective_end_date` is `NULL`, it means the plan is still active indefinitely. For the purpose of this monthly calculation, assume it continues through the end of the target month (March 31, 2023).
- Round the final `total_billed_amount` to 2 decimal places.

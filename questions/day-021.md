# SQL Scenario-Based Interview Question: Pro-Rated Subscription Billing Calculation

## Business Scenario
You are a Data Analyst at "Subscribr", a SaaS company offering various subscription plans. Customers can upgrade or downgrade their plans at any point during their billing cycle. The company needs to calculate accurate pro-rated charges for each customer for a given billing month, taking into account all plan changes that occurred within or overlapped with that month. This is crucial for monthly revenue recognition and customer billing statements. Your task is to calculate the total pro-rated charge for a specific customer for a specific month.

Assume:
```sql
CURRENT_DATE = '2023-10-26'
```

---

# Database Setup

## Create Customers Table
```sql
CREATE TABLE Customers (
    customer_id INT PRIMARY KEY,
    customer_name VARCHAR(100)
);
```

## Insert Customers Data
```sql
INSERT INTO Customers (customer_id, customer_name) VALUES
(1, 'Alice Smith'),
(2, 'Bob Johnson');
```

## Create SubscriptionPlans Table
```sql
CREATE TABLE SubscriptionPlans (
    plan_id INT PRIMARY KEY,
    plan_name VARCHAR(50),
    monthly_price DECIMAL(10, 2)
);
```

## Insert SubscriptionPlans Data
```sql
INSERT INTO SubscriptionPlans (plan_id, plan_name, monthly_price) VALUES
(101, 'Basic', 100.00),
(102, 'Pro', 200.00),
(103, 'Premium', 300.00);
```

## Create SubscriptionEvents Table
```sql
CREATE TABLE SubscriptionEvents (
    event_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT,
    plan_id INT,
    event_date DATE,
    FOREIGN KEY (customer_id) REFERENCES Customers(customer_id),
    FOREIGN KEY (plan_id) REFERENCES SubscriptionPlans(plan_id)
);
```

## Insert SubscriptionEvents Data
```sql
INSERT INTO SubscriptionEvents (customer_id, plan_id, event_date) VALUES
(1, 101, '2023-08-15'), -- Alice starts with Basic
(1, 102, '2023-09-01'), -- Alice upgrades to Pro
(1, 101, '2023-09-10'), -- Alice downgrades to Basic
(1, 102, '2023-09-25'), -- Alice upgrades to Pro again
(1, 103, '2023-10-01'), -- Alice upgrades to Premium
(2, 101, '2023-08-20'), -- Bob starts with Basic
(2, 102, '2023-09-15'); -- Bob upgrades to Pro
```

# Task
Calculate the total pro-rated subscription charge for `customer_id = 1` for the billing month of September 2023.
The daily cost for a plan should be calculated as `monthly_price / (number of days in the billing month)`.
The billing month should be considered from its first day to its last day.
Round the final charge to 2 decimal places.

---

# Expected Output Format
| customer_id | billing_month | total_charge |
|-------------|---------------|--------------|
| 1           | 2023-09       | 150.00       |

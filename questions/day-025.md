# SQL Scenario-Based Interview Question: SaaS Customer At-Risk Identification

## Business Scenario
A rapidly growing SaaS company is facing challenges with customer retention. While outright cancellations are tracked, the Customer Success team wants to proactively identify customers who show signs of "soft churn" or dissatisfaction before they fully cancel. They need a robust SQL query to identify "at-risk" customers each month, allowing them to intervene with targeted outreach. You are a Data Analyst tasked with building this reporting logic.

Assume:
```sql
CURRENT_DATE = '2023-10-26'
```

# Database Setup

## Create Customers Table
```sql
CREATE TABLE customers (
    customer_id INT PRIMARY KEY,
    customer_name VARCHAR(100) NOT NULL,
    signup_date DATE NOT NULL
);
```

## Create Plans Table
```sql
CREATE TABLE plans (
    plan_id INT PRIMARY KEY,
    plan_name VARCHAR(50) NOT NULL,
    monthly_cost DECIMAL(10, 2) NOT NULL,
    plan_level_rank INT NOT NULL UNIQUE -- 1 for Basic, 2 for Standard, 3 for Premium
);
```


## Create Subscriptions Table
```sql
CREATE TABLE subscriptions (
    subscription_id INT PRIMARY KEY,
    customer_id INT NOT NULL,
    plan_id INT NOT NULL,
    start_date DATE NOT NULL,
    end_date DATE, -- NULL if subscription is currently active
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id),
    FOREIGN KEY (plan_id) REFERENCES plans(plan_id)
);
```

# Task
Identify all "at-risk" customers for the *most recently completed full month* relative to `CURRENT_DATE`. An "at-risk" customer is defined by meeting *either* of the following conditions when comparing the most recently completed full month (e.g., September 2023 for `CURRENT_DATE = '2023-10-26'`) to the month immediately prior (e.g., August 2023):

1.  **Plan Downgrade**: The customer's active plan `plan_level_rank` for the most recent completed full month is lower than their active plan `plan_level_rank` for the month immediately prior.
    *   *Rule for active plan:* For any given month, consider the plan that was active on the *last day* of that month. If a customer changed plans multiple times within the month, the one active on the last day is the one that counts.
2.  **Significant Usage Drop**: The customer's *average daily active users* for the most recent completed full month dropped by more than 25% compared to the month immediately prior, *and* they did not upgrade their plan (i.e., they either stayed on the same plan level or downgraded).

Your query should return the `customer_name` and the `at_risk_reason` (which can be 'Downgrade', 'Usage Drop', or 'Both').

# SQL Scenario-Based Interview Question: Identifying "Early Dissatisfaction" Customers After Initial Purchase

## Business Scenario
An E-commerce company is keenly focused on improving customer retention and reducing churn. The analytics team hypothesizes that customers who exhibit a specific purchasing pattern are "at-risk": they make their first successful purchase, and then soon after (within 60 days), they make a second purchase that ultimately results in a return or refund. The company wants to identify these customers to understand their specific issues and intervene with targeted support or offers to prevent churn. As a Data Analyst, you need to provide a list of such unique customers.

Assume:
```sql
CURRENT_DATE = '2024-03-08'
```

---

# Database Setup

## Create Customers Table
```sql
CREATE TABLE Customers (
    customer_id INT PRIMARY KEY,
    customer_name VARCHAR(100)
);
```

## Insert Customers Data
```sql

(102, 'Bob Johnson'),
(103, 'Carol White'),
(104, 'David Green'),
(105, 'Eve Black'),
(106, 'Frank Brown'),
(107, 'Grace Lee');
```

## Create Orders Table
```sql
CREATE TABLE Orders (
    order_id INT PRIMARY KEY,
    customer_id INT,
    order_date DATE,
    order_status VARCHAR(50), -- 'Completed', 'Returned', 'Refunded', 'Cancelled', 'Pending'
    FOREIGN KEY (customer_id) REFERENCES Customers(customer_id)
);
```

# Task
Identify all unique `customer_id`s and their `customer_name`s who meet the following criteria:
1. They made their *first ever completed purchase* (an order with `order_status = 'Completed'`) on a specific `order_date`.
2. Subsequently, they made *at least one other purchase* within 60 days of that first completed purchase.
3. The `order_status` of *that subsequent purchase* (the one within 60 days) is either `'Returned'` or `'Refunded'`.

Return the `customer_id` and `customer_name` for these identified customers, ordered by `customer_id`.

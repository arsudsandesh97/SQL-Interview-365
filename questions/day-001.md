# SQL Scenario-Based Interview Question: Identifying Pre-Delivery Bottleneck Hubs

## Business Scenario
You are a Data Analyst at "SwiftParcel Logistics", a rapidly growing package delivery company. The Operations team is concerned about packages experiencing significant delays at intermediate hubs *before* they even reach the final mile for delivery. These pre-delivery delays directly impact customer satisfaction and operational efficiency. Your task is to analyze package tracking data to identify which packages accumulated substantial delays prior to delivery, and more critically, pinpoint the specific *hub* that contributed the largest individual delay for each such problematic package. This insight will help the operations team focus their resources on investigating and improving processes at identified bottleneck hubs.

Assume:
```sql
CURRENT_DATE = '2023-10-26'
```

## Create package_events Table
```sql
CREATE TABLE package_events (

    package_id VARCHAR(10) NOT NULL,

    event_type VARCHAR(20) NOT NULL, -- 'SCAN' for intermediate events, 'DELIVERED' for final
    event_timestamp TIMESTAMP NOT NULL,
    expected_timestamp TIMESTAMP NOT NULL
```

# Task
Write a SQL query that identifies packages with a **total pre-delivery delay of 90 minutes or more**. For each such package, you need to report its `package_id`, the `total_pre_delivery_delay_minutes`, and the `bottleneck_hub_id`. The `bottleneck_hub_id` is defined as the hub that recorded the *single largest individual delay* among all its intermediate (non-DELIVERED) events for that package. If there's a tie for the largest individual delay, any of the tied `hub_id`s is acceptable.

**Definitions:**
*   **Individual Delay:** `(event_timestamp - expected_timestamp)` for a specific event. Only positive delays (where `event_timestamp > expected_timestamp`) should be considered.
*   **Pre-delivery events:** All events for a package where `event_type` is 'SCAN'. Do NOT include the 'DELIVERED' event in the total pre-delivery delay calculation.

*(Note: For P003, HUB_Q would also be an acceptable bottleneck hub since it also had a 30-minute delay, tied with HUB_P for the largest individual delay for that package.)*

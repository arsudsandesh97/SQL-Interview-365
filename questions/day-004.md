# SQL Scenario-Based Interview Question: Analyzing Post-Return Customer Value

## Business Scenario
An e-commerce clothing retailer is struggling with high return rates and wants to better understand the true profitability of their customer base. Specifically, they are interested in customers who initially make returns but then go on to make subsequent purchases. The analytics team needs to calculate the *total net sales* generated by these "retained returners" for all their transactions that occurred *strictly after* their *first recorded return*. This analysis will help identify if these customers, despite initial returns, still contribute positively to revenue over time, guiding retention strategies.

Assume:
```sql
CURRENT_DATE = '2023-10-26'
```

---

# Database Setup

## Create CustomerTransactions Table
```sql
CREATE TABLE CustomerTransactions (
    transaction_id INT PRIMARY KEY,
    customer_id INT,
    order_id INT,
    transaction_date DATE,
    transaction_type VARCHAR(10), -- 'SALE' or 'RETURN'
    transaction_amount DECIMAL(10, 2) -- Positive for sales, negative for returns
);
```

# Task
Calculate the `total_net_sales` for each `customer_id` for all their transactions that occurred *strictly after* their *first recorded return transaction date*. Only include customers who have made at least one purchase (`transaction_type = 'SALE'`) *after* their first return date. The `total_net_sales` should reflect the sum of `transaction_amount` for all relevant post-first-return transactions.

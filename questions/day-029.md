# SQL Scenario-Based Interview Question: Identifying Long-Dwell Packages in Logistics Hubs

## Business Scenario
A major logistics company, "RapidRoute Logistics," aims to enhance its operational efficiency and customer satisfaction by proactively identifying packages that experience unusual delays within their sorting centers (hubs). They've observed that packages sometimes get "stuck" at a specific hub, meaning they remain there for an extended period between consecutive scans at the same location. This often leads to missed delivery windows and increased customer complaints.

Your role as a Data Analyst is to help RapidRoute Logistics pinpoint these "long-dwell" packages. Specifically, they need to identify packages that have been scanned consecutively at the *same hub*, where the time difference between these two consecutive scans exceeds a predefined threshold of 24 hours. They are interested in reporting only the *most recent* such "long-dwell" event for each package.

Assume:
```sql
CURRENT_DATE = '2023-11-20'
```

# Database Setup

## Create packages Table
```sql
CREATE TABLE packages (
    package_id VARCHAR(50) PRIMARY KEY,
    destination_city VARCHAR(100)
```

## Create package_scans Table
```sql
CREATE TABLE package_scans (
    scan_id INT PRIMARY KEY,
    package_id VARCHAR(50),

    scan_timestamp DATETIME,
    FOREIGN KEY (package_id) REFERENCES packages(package_id),
    FOREIGN KEY (hub_id) REFERENCES hubs(hub_id)
);
```

# Task
Identify packages that have been scanned consecutively at the *same hub*, where the time difference between the *previous scan* and the *current scan* for that package at that specific hub is greater than 24 hours. For each such package, report only the *most recent* instance of this "long-dwell" event.

Your output should include:
- `package_id`
- `hub_name` (where the long-dwell occurred)
- `city` (of the hub)
- `long_dwell_start_timestamp` (the timestamp of the earlier scan in the long-dwell pair)
- `long_dwell_end_timestamp` (the timestamp of the later scan in the long-dwell pair)
- `dwell_duration_hours` (the calculated duration in hours between the two scans, rounded to 2 decimal places)

# SQL Scenario-Based Interview Question: Identifying At-Risk Recent Purchasers for Re-engagement

## Business Scenario
You are a Data Analyst at "ShopSmart," an e-commerce company. The marketing team wants to launch a re-engagement campaign targeting customers who have made a recent purchase but have shown no subsequent activity on the website (like browsing products or adding items to their cart) within a critical 7-day window after their last order. They consider these customers "at-risk" of becoming inactive. Your task is to identify these customers, along with their last purchase date, so the marketing team can send them personalized offers.

Assume:
```sql
CURRENT_DATE = '2023-10-26'
```

---

# Database Setup

## Create customers Table
```sql
CREATE TABLE customers (
    customer_id VARCHAR(50) PRIMARY KEY,
    customer_name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL
);
```

## Insert customers Data
```sql
INSERT INTO customers (customer_id, customer_name, email) VALUES
('C101', 'Alice Wonderland', 'alice.w@example.com'),
('C102', 'Bob The Builder', 'bob.t@example.com'),
('C103', 'Charlie Chaplin', 'charlie.c@example.com'),
('C104', 'David Copperfield', 'david.c@example.com'),
('C105', 'Eve Harrington', 'eve.h@example.com');
```

## Create orders Table
```sql
CREATE TABLE orders (
    order_id VARCHAR(50) PRIMARY KEY,
    customer_id VARCHAR(50) NOT NULL,
    order_date DATE NOT NULL,
    total_amount DECIMAL(10, 2) NOT NULL,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);
```

## Insert orders Data
```sql

('ORD001', 'C101', '2023-10-20', 120.00),
('ORD002', 'C102', '2023-10-10', 55.50),
('ORD003', 'C103', '2023-09-01', 200.00), -- Old purchase
('ORD004', 'C104', '2023-10-22', 75.20),
('ORD005', 'C105', '2023-10-19', 30.00),
('ORD006', 'C101', '2023-09-28', 90.00); -- Alice's previous order
```

## Create user_activity Table
```sql
CREATE TABLE user_activity (
    activity_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id VARCHAR(50) NOT NULL,
    activity_type VARCHAR(50) NOT NULL, -- e.g., 'PRODUCT_VIEW', 'ADD_TO_CART', 'CHECKOUT_COMPLETE'
    activity_timestamp DATETIME NOT NULL,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);
```

## Insert user_activity Data

# Task
Write a SQL query to identify all customers who meet the following criteria:
1.  They made their *latest* purchase within the last 30 days from `CURRENT_DATE`.
2.  After their latest purchase, they have shown *no engagement* (i.e., no 'PRODUCT_VIEW' or 'ADD_TO_CART' activity) within 7 days *after* that latest purchase date.

The query should return the `customer_id`, `customer_name`, `email`, and the `last_purchase_date` for these "at-risk" customers.

# SQL Scenario-Based Interview Question: Subscription Value Migration Analysis

## Business Scenario
You are a Data Analyst at a fast-growing SaaS company that offers tiered subscription plans (Basic, Pro, Premium). The leadership team wants to understand customer movement between these plans and identify their most loyal customers. They are particularly interested in recent changes to track the impact of new pricing strategies and marketing efforts. Your task is to provide a comprehensive view of customer subscription activities.

Assume:
```sql
CURRENT_DATE = '2023-10-15'
```

---

# Database Setup

## Create Plans Table
```sql
CREATE TABLE Plans (
    plan_id INT PRIMARY KEY,
    plan_name VARCHAR(50) NOT NULL,
    monthly_price DECIMAL(10, 2) NOT NULL
);
```

## Insert Plans Data
```sql
INSERT INTO Plans (plan_id, plan_name, monthly_price) VALUES
(1, 'Basic', 10.00),
(2, 'Pro', 25.00),
(3, 'Premium', 50.00);
```

## Create Subscriptions Table
```sql
CREATE TABLE Subscriptions (
    subscription_id SERIAL PRIMARY KEY,
    user_id VARCHAR(50) NOT NULL,
    plan_id INT NOT NULL REFERENCES Plans(plan_id),
    start_date DATE NOT NULL,
    end_date DATE, -- NULL if current/active subscription
    CONSTRAINT unique_subscription_per_user_date UNIQUE (user_id, start_date)
);
```

# Task

Generate a report that identifies two distinct groups of customers:

1.  **Customers with Recent Subscription Changes (Last Full Calendar Month):**
    Identify users whose subscription status changed (upgraded, downgraded, or churned) within the last *full calendar month* relative to `CURRENT_DATE`.
    *   **Upgrade:** Switched to a higher-priced plan.
    *   **Downgrade:** Switched to a lower-priced plan.
    *   **Churn:** Their last active subscription ended within this period, and they do not have an active subscription as of `CURRENT_DATE`.

2.  **Loyal Customers:**
    Identify users who have been on the *exact same plan continuously* for at least the *last 6 consecutive months* ending on `CURRENT_DATE`, and are *still active* on that plan. "Continuously" means no gaps and no plan changes during this 6-month period.

The output should consolidate these groups, showing `user_id`, the `event_type` (`Upgrade`, `Downgrade`, `Churn`, `Loyal Customer`), and `plan_name_relevant` (the plan involved in the change for recent events, or the current plan for loyal customers). A user might appear in only one category. If a user qualifies for both "recent change" and "loyal customer", prioritize the "recent change" category.

**Reference Dates:**
*   For `CURRENT_DATE = '2023-10-15'`:
    *   The "last full calendar month" is September 2023 (from '2023-09-01' to '2023-09-30').
    *   The "last 6 consecutive months ending on `CURRENT_DATE`" refers to the period from '2023-04-15' to '2023-10-15'.

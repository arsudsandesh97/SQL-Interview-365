# SQL Scenario-Based Interview Question: Identify High-Risk Delivery Zones by Consecutive Late Deliveries

## Business Scenario
A large e-commerce logistics company, "FastShip," aims to improve its delivery efficiency and customer satisfaction. The operations team has noticed anecdotal evidence of recurring delays in certain geographic zones but lacks concrete data to identify and prioritize these "high-risk" areas. As a Data Analyst, you've been tasked with providing insights to pinpoint delivery zones that are consistently experiencing late deliveries, indicating potential underlying issues such as traffic bottlenecks, driver route inefficiencies, or capacity problems.

Your goal is to identify delivery zones that have shown a recent trend of consecutive late deliveries, specifically within the last week, so that management can deploy resources or investigate further.

Assume:
```sql
CURRENT_DATE = '2023-10-26'
```

---

# Database Setup

## Create Deliveries Table
```sql
CREATE TABLE Deliveries (
    delivery_id INT PRIMARY KEY,
    order_id INT,
    delivery_zone_id INT,
    delivery_date DATE,
    scheduled_delivery_timestamp DATETIME,
    actual_delivery_timestamp DATETIME
);
```

# Task
Identify all `delivery_zone_id`s that have experienced a streak of **at least 3 consecutive late deliveries** within the last 7 days from `CURRENT_DATE`. For each identified zone, report the `delivery_zone_id` and the `MAXIMUM` number of consecutive late deliveries observed for that zone during this 7-day period.

A delivery is considered 'late' if `actual_delivery_timestamp` is later than `scheduled_delivery_timestamp`.
The 7-day period should include `CURRENT_DATE`. So for `CURRENT_DATE = '2023-10-26'`, the period is from `2023-10-20` to `2023-10-26` (inclusive).

Order the results by `max_consecutive_late_deliveries` in descending order, then by `delivery_zone_id` in ascending order.

---

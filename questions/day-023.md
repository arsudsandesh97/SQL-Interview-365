# SQL Scenario-Based Interview Question: Identify Underutilized Premium SaaS Subscribers with High Engagement

## Business Scenario
A leading SaaS company offers various subscription tiers, including 'Standard' and 'Premium' plans, with differing limits on key features like project capacity (Feature A) and storage (Feature B). The sales and customer success teams are looking for ways to proactively manage churn risk and optimize customer subscriptions. They are particularly interested in identifying "underutilized" Premium subscribers who might be paying for more than they need, but are still actively using the platform. These customers represent an opportunity for either a proactive tier downgrade recommendation to prevent churn (if engagement is low and usage consistently below tier), or an upsell engagement strategy (if engagement is high, suggesting potential to leverage more features).

Your task as a Data Analyst is to identify active Premium subscribers who consistently utilize features at a level typically associated with the 'Standard' plan, while still maintaining a healthy level of engagement (login activity).

Assume:
```sql
CURRENT_DATE = '2023-11-15'
```

# Database Setup

## Create customers Table
```sql
CREATE TABLE customers (
    customer_id VARCHAR(50) PRIMARY KEY,
    customer_name VARCHAR(100),
    created_at DATE
);
```

## Create plans Table
```sql
CREATE TABLE plans (
    plan_id VARCHAR(50) PRIMARY KEY,
    plan_name VARCHAR(100),
    monthly_cost DECIMAL(10, 2),
    feature_a_limit INT, -- e.g., max projects
    feature_b_limit INT  -- e.g., max storage in GB
);
```

## Create subscriptions Table
```sql
CREATE TABLE subscriptions (
    subscription_id VARCHAR(50) PRIMARY KEY,
    customer_id VARCHAR(50),

    end_date DATE,
    status VARCHAR(50),
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id),
    FOREIGN KEY (plan_id) REFERENCES plans(plan_id)
);
```

# Task
Identify `customer_id` and `customer_name` for all active 'Premium' subscribers who meet *all* of the following conditions:
1.  Their current active subscription is for the 'Premium' plan.
2.  For each of the last three *full* calendar months (based on `CURRENT_DATE = '2023-11-15'`, these are August, September, and October 2023), their total monthly usage for `feature_a_used` was less than or equal to the `feature_a_limit` of the 'Standard' plan.
3.  For each of the last three *full* calendar months, their total monthly usage for `feature_b_used` was less than or equal to the `feature_b_limit` of the 'Standard' plan.
4.  For each of the last three *full* calendar months, they had at least 15 distinct login days.

The output should be sorted by `customer_id`.

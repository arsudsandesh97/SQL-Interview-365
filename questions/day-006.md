# SQL Scenario-Based Interview Question: Monthly SaaS Subscription Snapshot

## Business Scenario
You are a Data Analyst at a growing SaaS company that offers various subscription plans. Your team is responsible for tracking key subscription metrics to understand customer lifecycle, growth, and churn. The marketing and finance teams rely on accurate monthly reports to gauge performance and make strategic decisions. You need to generate a report that shows the number of new trials started, the count of active paid subscriptions, and the total Monthly Recurring Revenue (MRR) from these active paid subscriptions at the end of each month.

Assume:
```sql
CURRENT_DATE = '2023-04-01'
```

---

# Database Setup

## Create plans Table
```sql
CREATE TABLE plans (
    plan_id INT PRIMARY KEY,
    plan_name VARCHAR(50) NOT NULL,
    mrr_value DECIMAL(10, 2) NOT NULL,
    is_trial BOOLEAN NOT NULL
);
```

## Insert plans Data
```sql
INSERT INTO plans (plan_id, plan_name, mrr_value, is_trial) VALUES
(1, 'Free Trial', 0.00, TRUE),
(2, 'Basic Monthly', 25.00, FALSE),
(3, 'Pro Monthly', 50.00, FALSE),
(4, 'Enterprise Monthly', 100.00, FALSE);
```

## Create subscription_events Table
```sql
CREATE TABLE subscription_events (
    event_id INT PRIMARY KEY,
    customer_id VARCHAR(10) NOT NULL,
    event_type VARCHAR(50) NOT NULL, -- e.g., 'trial_started', 'converted_to_paid', 'plan_upgraded', 'plan_downgraded', 'cancelled'
    event_date DATE NOT NULL,
    plan_id_after_event INT NOT NULL, -- The plan active *after* this event occurred
    FOREIGN KEY (plan_id_after_event) REFERENCES plans(plan_id)
);
```

# Task
Generate a monthly report for the period from January 2023 to March 2023 (inclusive). For each month, you need to calculate:

1.  `report_month`: The first day of the month (e.g., '2023-01-01').
2.  `new_trials_started`: The count of unique customers who initiated their *first* trial (`event_type = 'trial_started'`) during that specific month.
3.  `active_paid_subscriptions_eom`: The count of unique customers who had an *active paid subscription* at the *end of that month*. A subscription is considered "active paid" at the end of a month if:
    *   The customer had an event on or before the end of the report month.
    *   The *latest* event for that customer on or before the end of the report month was *not* 'cancelled'.
    *   The `plan_id_after_event` associated with that *latest* event is a `is_trial = FALSE` plan.
4.  `eom_mrr`: The total Monthly Recurring Revenue (MRR) from these `active_paid_subscriptions_eom` at the *end of that month*. The MRR should correspond to the `mrr_value` of the *final paid plan* each qualifying customer was on at the end of the month.

The report should include a row for every month in the specified range, even if some counts are zero.

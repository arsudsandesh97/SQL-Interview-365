# SQL Scenario-Based Interview Question: Credit Card Spending Anomaly Detection

## Business Scenario
A major financial institution's fraud detection team needs to identify credit card accounts that are exhibiting unusual spikes in transaction volume. A sudden and significant increase in the number of transactions might indicate potential fraudulent activity or a change in customer behavior that requires further investigation. As a Data Analyst, your task is to pinpoint such accounts by comparing recent spending patterns against historical averages.

Assume:
```sql
CURRENT_DATE = '2023-11-15'
```

---

# Database Setup

## Create transactions Table
```sql
CREATE TABLE transactions (
    transaction_id INT PRIMARY KEY,
    account_id INT NOT NULL,
    transaction_date DATE NOT NULL,
    amount DECIMAL(10, 2) NOT NULL
);
```

# Task
Write a SQL query to identify `account_id`s that exhibit a significant surge in transaction volume. An account is considered to have a significant surge if its total number of transactions in the most recent 7-day period (inclusive, ending on `CURRENT_DATE`) is at least double its average weekly transaction count over the 90-day period immediately preceding the recent 7-day period.

Your query should return a single column: `account_id`.

---

### Result

|---|
| 102 |
| 104 |

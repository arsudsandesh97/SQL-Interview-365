# SQL Scenario-Based Interview Question: Identifying "Browse-Then-Buy" Conversion Events

## Business Scenario
An e-commerce company is meticulously analyzing customer behavior to understand the direct impact of product page views on immediate sales. They are particularly interested in identifying specific instances where a customer viewed a particular product and subsequently purchased *that exact same product* within a 24-hour window from the view event. This analysis is crucial for prioritizing product page optimizations, evaluating content effectiveness, and understanding rapid conversion paths. As a Data Analyst, your task is to extract a clear list of customer-product pairs that demonstrate this "browse-then-buy" behavior.

Assume:
```sql
CURRENT_DATE = '2023-10-26'
```

---

# Database Setup

## Create customer_events Table
```sql
CREATE TABLE customer_events (
    event_id INTEGER PRIMARY KEY,
    customer_id INTEGER,
    product_id INTEGER,
    event_type TEXT, -- e.g., 'view_product', 'purchase_product', 'add_to_cart'
    event_timestamp TEXT -- Stored as 'YYYY-MM-DD HH:MM:SS'
);
```

# Task
Identify each unique customer-product pair that demonstrates a "browse-then-buy" behavior. A "browse-then-buy" behavior is defined as a customer viewing a specific product (`event_type = 'view_product'`) and subsequently purchasing *that exact same product* (`event_type = 'purchase_product'`) within a 24-hour window from the view event.

For each such unique customer-product pair, you should report:
1. `customer_id`
2. `product_id`
3. The `event_timestamp` of the *earliest purchase* that satisfied the condition (i.e., had *any* preceding view for the same product within 24 hours).
